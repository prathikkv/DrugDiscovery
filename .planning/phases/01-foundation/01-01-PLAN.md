  <action>
    Create the authentication module implementing REQ-501 (auth + RBAC) and REQ-502 (lockout).

    1. **src/auth/__init__.py**: Export `AuthService`, `User`, `Role`.

    2. **src/auth/models.py**:
       - `Role` enum with values: `ADMIN = "admin"`, `ANALYST = "analyst"`, `REVIEWER = "reviewer"`
       - `User` dataclass with fields: `user_id: str`, `email: str`, `role: Role`, `created_at: str`, `is_active: bool`, `failed_attempts: int`, `locked_until: Optional[str]`
       - Do NOT include password_hash in the User dataclass (security -- never expose hash outside auth module)

    3. **src/auth/db.py**:
       - `AUTH_SCHEMA` string with CREATE TABLE for `users`:
         - `user_id TEXT PRIMARY KEY` (UUID)
         - `email TEXT UNIQUE NOT NULL`
         - `password_hash TEXT NOT NULL` (bcrypt hash)
         - `role TEXT NOT NULL CHECK(role IN ('admin', 'analyst', 'reviewer'))`
         - `created_at TEXT NOT NULL` (ISO 8601 UTC)
         - `is_active INTEGER NOT NULL DEFAULT 1`
         - `failed_attempts INTEGER NOT NULL DEFAULT 0`
         - `locked_until TEXT` (ISO 8601 UTC or NULL)
       - `init_auth_db(conn)` -- execute schema DDL
       - `insert_user(conn, user_id, email, password_hash, role, created_at)` -- INSERT with parameterized query
       - `get_user_by_email(conn, email)` -- SELECT returning Row or None
       - `get_user_by_id(conn, user_id)` -- SELECT returning Row or None
       - `update_failed_attempts(conn, user_id, attempts, locked_until=None)` -- UPDATE with BEGIN IMMEDIATE for atomicity (see RESEARCH.md Pitfall 7)
       - `reset_failed_attempts(conn, user_id)` -- SET failed_attempts=0, locked_until=NULL

    4. **src/auth/service.py**:
       - `AuthService` class:
         - `__init__(self, db_path: Path = None)` -- defaults to `config.AUTH_DB`, calls `init_auth_db`
         - Uses `threading.Lock` for write serialization (see RESEARCH.md Pitfall 1)
         - `register(email, password, role) -> dict` -- validates email uniqueness, enforces password <= 72 bytes, hashes with `bcrypt.hashpw(password.encode("utf-8"), bcrypt.gensalt(rounds=config.BCRYPT_ROUNDS))`, generates UUID, stores with ISO 8601 UTC timestamp via `datetime.now(timezone.utc).isoformat()`. Returns `{"success": True, "user_id": ...}` or `{"success": False, "error": ...}`.
         - `login(email, password) -> dict` -- checks lockout status first (if locked_until is in the future, reject with remaining minutes). If credentials valid, reset failed_attempts and return `{"success": True, "user_id": ..., "role": ...}`. If invalid, increment failed_attempts. If failed_attempts >= LOCKOUT_THRESHOLD, set locked_until to now + LOCKOUT_DURATION_MINUTES. Return `{"success": False, "error": "Invalid credentials"}` (never reveal whether email exists).
         - `get_user(user_id) -> Optional[User]` -- lookup by ID, return User dataclass or None
         - Error messages MUST NOT distinguish between "email not found" and "wrong password" (security best practice)
       - Each method creates its own connection via `get_connection(self.db_path)` (see RESEARCH.md Pitfall 1 -- separate connections per operation, not shared)
       - Use `bcrypt.checkpw()` for constant-time password comparison
  </action>
