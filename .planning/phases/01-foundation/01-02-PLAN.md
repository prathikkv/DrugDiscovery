    3. **src/compliance/audit_trail.py**:
       - `AuditTrail` class:
         - `__init__(self, db_path: Path = None)` -- defaults to `config.AUDIT_DB`, calls `init_audit_db`
         - Uses `threading.Lock` for serializing the read-previous-hash + compute-hash + insert sequence (see RESEARCH.md Pitfall 2 -- hash chain ordering)
         - `_compute_record_hash(previous_hash, timestamp, user_id, action, resource_type, resource_id, details) -> str`:
           - Build a dict with all fields, use `json.dumps(sort_keys=True, separators=(",", ":"))` for deterministic serialization
           - Return `hashlib.sha256(payload.encode("utf-8")).hexdigest()`
         - `append_record(user_id, action, resource_type, resource_id, details=None) -> str`:
           - Acquire lock
           - Create new connection (thread safety)
           - Read previous hash: `SELECT record_hash FROM audit_trail ORDER BY sequence_id DESC LIMIT 1` -- if no rows, use genesis hash `"0" * 64`
           - Generate timestamp: `datetime.now(timezone.utc).isoformat()`
           - Compute record_hash via `_compute_record_hash`
           - INSERT with all fields
           - Commit and close connection
           - Release lock
           - Return the record_hash
         - `verify_chain() -> dict`:
           - Read ALL records ordered by sequence_id ASC
           - For each record: verify previous_hash matches expected, recompute record_hash and verify it matches stored
           - Return `{"valid": bool, "records_checked": int, "first_broken": Optional[int], "error": Optional[str]}`
         - `get_records(user_id=None, resource_type=None, resource_id=None, limit=100) -> list[dict]`:
           - Query with optional filters, return list of dicts
         - CRITICAL: NEVER expose UPDATE or DELETE methods on audit_trail. This is append-only by design. No method should ever modify or remove existing records.

    4. **src/compliance/electronic_signature.py** (REQ-504):
       - `ElectronicSignature` class:
         - `__init__(self, audit_trail: AuditTrail, auth_db_path: Path = None)` -- takes audit trail instance for recording, auth_db_path defaults to config.AUTH_DB
         - `sign(user_id, password, resource_type, resource_id, meaning) -> dict`:
           - **Re-authenticate**: Use `get_connection(self.auth_db_path)` to look up user by user_id, then `bcrypt.checkpw()` to verify password. If re-auth fails, return `{"success": False, "error": "Re-authentication failed"}`.
           - Generate timestamp: `datetime.now(timezone.utc).isoformat()`
           - Build signature payload dict: `{"user_id", "timestamp", "resource_type", "resource_id", "meaning"}`
           - Compute `signature_hash = hashlib.sha256(json.dumps(payload, sort_keys=True, separators=(",", ":")).encode()).hexdigest()`
           - Record in audit trail via `self.audit_trail.append_record(user_id, "SIGN", resource_type, resource_id, details={"meaning": meaning, "signature_hash": signature_hash})`
           - Return `{"success": True, "signature_hash": ..., "timestamp": ...}`
         - Import bcrypt directly for re-authentication (do NOT import AuthService to avoid circular dependency -- just do the password check inline)

           - Create `ThreadPoolExecutor(max_workers=max_workers)`
           - Use `threading.Lock` for write serialization
           - Initialize schema via `init_tasks_db`
         - `submit(task_id, task_type, fn, *args, project_id=None, **kwargs) -> str`:

         - `_wrapped_run(self, task_id, fn, *args, **kwargs)`:
           - Update status to RUNNING with started_at timestamp
           - Call `fn(*args, **kwargs)` -- the fn receives a `progress_callback` kwarg if it accepts one
           - On success: update status to COMPLETED with completed_at and result_json
           - On exception: update status to FAILED with error_message and completed_at
           - Each database write uses its own connection (thread safety)
         - `update_progress(task_id, progress: float, checkpoint_path: str = None)`:

         - `get_status(task_id) -> Optional[TaskRecord]`:
           - Read from SQLite, return TaskRecord or None

         - `get_tasks(project_id=None, status=None) -> list[TaskRecord]`:
           - Query with optional filters
         - `cancel(task_id) -> bool`:
           - Set status to CANCELLED if currently PENDING (cannot cancel RUNNING in ThreadPoolExecutor without cooperation)
         - `shutdown(wait=True)`:
           - Call `self.executor.shutdown(wait=wait)`
         - IMPORTANT: Each database operation creates its own connection via `get_connection(self.db_path)`. Do NOT store a connection as an instance variable -- threads will use different connections.
