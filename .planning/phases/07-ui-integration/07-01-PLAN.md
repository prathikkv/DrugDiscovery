On success, sets session_state gate flag and calls st.rerun(). On failure, shows st.error.

IMPORTANT: All widget keys must be prefixed to avoid collisions. Use `f"hitl_{gate_id}_..."` pattern.

IMPORTANT: The dialog trigger must use session_state flags (not button state) to survive st.rerun(). When Approve/Reject is clicked, set `st.session_state[f"show_esign_{gate_id}"] = True` and `st.session_state[f"esign_action_{gate_id}"] = approved`. Then check this flag OUTSIDE the button block to call the dialog function.
