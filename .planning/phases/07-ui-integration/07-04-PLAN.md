For each of the 6 scenarios, generate 4 JSON files. The data must be scientifically plausible and tell a compelling story for a pharma VP demo. Use the existing model schemas from the codebase to ensure compatibility.

**evidence.json** structure (matches serialized AggregatedEvidence):
```json
{
  "gene": {
    "canonical_symbol": "EGFR",
    "ensembl_id": "ENSG00000146648",
    "uniprot_accession": "P00533",
    "query_symbol": "EGFR"
  },
  "disease_context": "Non-Small Cell Lung Cancer",
  "results": {
    "opentargets": {
      "confidence": 1.0,
      "data": { "associations": [...], "overall_score": 0.9 },
      "error": null,
      "is_fallback": false
    },
    "dgidb": { ... },
    "pubmed": { ... },
    "clinicaltrials": { ... },
    "uniprot": { ... },
    "chembl": { ... }
  }
}
```

**scoring.json** structure (matches ScorecardResult.model_dump()):
Must include composite (score, dimension_scores with 7 DimensionScore dicts each having name, score, max_score, data_coverage, sub_scores), verdict (level, rationale, forced_conditional, dimension_violations), gene_symbol, disease_context, evidence_hash, scored_at.

**reasoning.json** structure (maps mode_name -> ReasoningResult.model_dump()):
5 entries: hypothesis, synthesis, contradiction, gap, confidence. Each with mode, gene_symbol, disease_context, claims (list of {text, confidence, sources}), summary, raw_output, tool_trace (can be minimal), hallucination_issues.

**pipeline_report.json** structure:
Minimal pipeline report with QC metrics, cell counts, gene counts, tissue type. No real UMAP data needed -- just enough to display in the omics page.

**Scenario-specific data requirements**:

1. **EGFR/NSCLC** (AstraZeneca): GO verdict, composite >=75. Strong across all dimensions. 15+ disease associations in OpenTargets. Drugs: erlotinib, gefitinib, osimertinib, afatinib. 50+ clinical trials. Rich PubMed (200+ papers). Known kinase with crystal structures. NSCLC-specific mutations (L858R, T790M, C797S).

2. **ESR1/ER+Breast** (AstraZeneca): GO verdict, composite >=70. Strong genetic evidence and clinical validation. Drugs: tamoxifen, fulvestrant, elacestrant. Mention ESR1 mutations in endocrine resistance. 30+ trials.

3. **PIK3CA/HR+Breast** (Roche): GO verdict, composite >=68. PI3K pathway mutations (~40% of HR+ breast cancers). Drug: alpelisib (Piqray). Note toxicity challenges (hyperglycemia). Moderate competitive landscape.

4. **GLP1R/Obesity** (Eli Lilly): GO verdict, composite >=72. Breakthrough metabolic target. Drugs: semaglutide, tirzepatide, liraglutide. Massive market but crowded competitive landscape. Strong safety profile.

5. **PARP1/BRCA+Breast** (Merck): GO verdict, composite >=70. Synthetic lethality in BRCA-mutant cancers. Drugs: olaparib (Lynparza), niraparib, rucaparib, talazoparib. Strong mechanism but resistance challenges.

6. **CD274/Pan-cancer** (Merck): CONDITIONAL verdict, composite 55-70. PD-L1 is validated but extremely competitive (crowded immuno-oncology space). Drugs: pembrolizumab, atezolizumab, durvalumab, avelumab, nivolumab (anti-PD-1). Moderate competitive landscape score to force CONDITIONAL. At least one dimension violation.

For each scenario:
- evidence.json: All 6 sources should have confidence > 0.0 (realistic for established targets). Include 3-5 associations for OpenTargets, 2-4 drug interactions for DGIdb, 5-10 papers for PubMed, 3-8 trials for ClinicalTrials, protein info for UniProt, compound data for ChEMBL.
- scoring.json: Composite score must match the verdict criteria. Each dimension has a score (0-1 normalized to 0-max_score). Include 2-4 sub_scores per dimension.
- reasoning.json: 3-5 claims per mode with realistic scientific language and source citations in [Source: SourceName] format.
- pipeline_report.json: QC metrics (cells: 2000-8000, genes: 15000-25000, mito_pct: 5-15%, tissue_type), minimal DE summary.

Use `json.dumps(data, indent=2, default=str)` for readability.

After generating the script, RUN it to create all 24 files:

