**src/pipeline/processing.py:** Create `run_processing(adata, config, project_dir, progress_tracker=None) -> tuple[AnnData, dict]` refactored from bioorchestrator_real/pipeline/stage4_process.py. Key implementation:

1. Check for existing checkpoint via `load_checkpoint(project_dir, "processing")` -- return early if found.
2. Store raw counts: `adata.layers["counts"] = adata.X.copy()` (needed for CellTypist and DE later).
3. Normalize: `sc.pp.normalize_total(adata, target_sum=config.processing.normalize_total_target)` then `sc.pp.log1p(adata)`.
4. HVG selection: `sc.pp.highly_variable_genes(adata, n_top_genes=config.processing.n_top_hvgs, batch_key=batch_key_if_present, flavor="seurat_v3" if batch_key else "seurat", layer="counts" if batch_key else None)`.
5. Subset to HVGs for PCA: `adata_hvg = adata[:, adata.var["highly_variable"]].copy()`, scale with max_value=10, PCA with n_comps=config.processing.n_pcs, svd_solver="arpack", random_state=42.
6. Transfer PCA back: `adata.obsm["X_pca"] = adata_hvg.obsm["X_pca"]`, delete adata_hvg, gc.collect().
7. Harmony batch correction: if batch_key column exists in adata.obs and has >1 unique value, run `harmonypy.run_harmony()` with random_state=42. Store in obsm["X_pca_harmony"]. Use X_pca_harmony for neighbors if available, else X_pca. Wrap in try/except -- fall back to X_pca on failure.
8. Neighbors: `sc.pp.neighbors(adata, use_rep=use_rep, n_neighbors=config.processing.n_neighbors, n_pcs=min(config.processing.n_pcs_neighbors, shape[1]), random_state=42)`.
9. UMAP: `sc.tl.umap(adata, random_state=42)`.
10. Leiden: `sc.tl.leiden(adata, resolution=config.processing.leiden_resolution, random_state=42)`.
11. Set sc.settings.verbosity = 0 at start. Suppress warnings during PCA and UMAP with `warnings.catch_warnings()`.
12. Save checkpoint: `save_checkpoint(adata, project_dir, "processing")`.
13. Update progress if tracker provided.
14. Return (adata, processing_report) where report includes: stage="processing", n_hvgs, n_pcs, batch_corrected (bool), n_clusters, use_rep, timestamp.

NO console.print, NO rich imports, NO sc.settings.figdir. This is a pure computation module.

**src/pipeline/ambient_rna.py:** Create `run_ambient_rna_removal(adata, config, project_dir, progress_tracker=None, raw_counts_path=None) -> tuple[AnnData, dict]`:

1. If config.skip_ambient_rna is True, return (adata, {"stage": "ambient_rna", "status": "skipped", "reason": "user_skip", "warning": "Ambient RNA removal was skipped by user configuration"}).
2. Check for checkpoint first.
3. Try SoupX via rpy2:
   - Attempt `import rpy2.robjects as ro; from rpy2.robjects.packages import importr; soupx = importr("SoupX")`
   - If import fails, return skip report with reason="soupx_unavailable" and warning message.
   - SoupX needs raw (unfiltered) counts. If raw_counts_path is None or doesn't exist, return skip report with reason="no_raw_counts" and warning="SoupX requires unfiltered (raw) count matrix. Only filtered data provided."
   - If raw counts available: load raw counts, convert matrices to R format, run SoupChannel -> setClusters -> autoEstCont -> adjustCounts, convert back to Python AnnData.
4. On any exception during SoupX execution, catch it and return skip report with the error message.
5. Save checkpoint on success.
6. Report includes: stage="ambient_rna", status="completed"|"skipped", method="soupx"|"skipped", reason (if skipped), contamination_fraction (if completed), warning (if skipped).

IMPORTANT: The HITL gate for SoupX skip is a UI concern (Phase 7). At the pipeline level, the skip decision is encoded in config.skip_ambient_rna or detected automatically (no raw counts). The audit trail logging of the skip happens in the orchestrator (Plan 03), not here.

**src/pipeline/annotation.py:** Create two main functions:

`run_annotation(adata, config, project_dir, progress_tracker=None) -> tuple[AnnData, dict]`:
1. Check for existing checkpoint.
2. Determine CellTypist model: if config.celltypist_model is set, use it directly. Otherwise, look up from TISSUE_MODEL_MAP using config.tissue_type, falling back to "Immune_All_Low.pkl".
3. Import celltypist, download models (force_update=False), load the selected model.
4. IMPORTANT: CellTypist requires log1p-normalized data at 10K counts/cell. Verify adata.X is normalized (check if adata.X.max() < 50 -- raw counts would be in hundreds/thousands). If not normalized, raise ValueError with message about normalization requirement.
5. Run `celltypist.annotate(adata, model=model, majority_voting=True, over_clustering="leiden")`. Wrap in warnings.catch_warnings().
6. Convert predictions: `adata = predictions.to_adata()`. Standardize: if "majority_voting" in obs.columns, set adata.obs["cell_type"] = adata.obs["majority_voting"]; elif "predicted_labels", use that.
7. Save checkpoint.
8. Update progress.
9. Return (adata, annotation_report) where report includes: stage="annotation", model_used, n_cell_types, cell_type_counts dict, timestamp.

`validate_annotations(adata, marker_dict=None, label_col="cell_type") -> list[dict]` (REQ-107):
1. Use a default CANONICAL_MARKERS dict if marker_dict is None:
   - "T cells": ["CD3D", "CD3E", "CD3G"]
   - "B cells": ["CD19", "MS4A1", "CD79A"]
   - "Macrophages": ["CD68", "CD163", "MRC1"]
   - "Endothelial": ["PECAM1", "VWF", "CDH5"]
   - "Fibroblasts": ["COL1A1", "COL3A1", "DCN"]
   - "Epithelial": ["EPCAM", "KRT8", "KRT18"]
   - "NK cells": ["NKG7", "GNLY", "KLRD1"]
   - "Monocytes": ["CD14", "FCGR3A", "S100A8"]
   - "Dendritic": ["FCER1A", "CD1C", "CLEC10A"]
   - "Plasma cells": ["JCHAIN", "MZB1", "SDC1"]
2. For each cell type in marker_dict, find cells annotated with a label containing that cell type name (case-insensitive str.contains).
3. For matching cells, check what percentage express each canonical marker (>0 in X).
4. If <10% express a canonical marker, flag as discrepancy: {"cell_type": str, "marker": str, "pct_expressing": float, "flag": "LOW_MARKER_EXPRESSION", "message": str}.
5. Handle sparse matrices (check issparse, toarray if needed).
6. Return list of discrepancy dicts.

**src/pipeline/differential_expression.py:** Create `run_differential_expression(adata, config, project_dir, progress_tracker=None) -> tuple[AnnData, dict]`:

1. Check for checkpoint.
2. Determine groupby column: use "cell_type" if it exists, else "leiden".
3. Filter groups with <20 cells -- log which groups were excluded in the report.
4. If no valid groups remain, return (adata, {"stage": "de", "status": "error", "reason": "No groups with >=20 cells"}).
5. Run `sc.tl.rank_genes_groups(adata_sub, groupby=groupby, method=config.de_method, corr_method=config.de_corr_method, n_genes=500, use_raw=False, pts=True)`. Use layer="counts" if "counts" in adata.layers.
6. Extract results per group using `sc.get.rank_genes_groups_df(adata_sub, group=group)` for each valid group.
7. Store structured results in adata.uns["de_results"] as a dict of {group_name: list_of_gene_dicts} where each gene dict has: names, scores, pvals, pvals_adj, logfoldchanges, pct_nz_group, pct_nz_reference.
8. Save checkpoint (save adata with results in .uns).
9. Update progress.
10. Return (adata, de_report) where report includes: stage="de", method, corr_method, groupby, n_groups_tested, groups_excluded (list with reasons), n_genes_per_group (dict), timestamp.
