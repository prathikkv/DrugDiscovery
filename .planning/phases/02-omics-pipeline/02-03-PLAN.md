**src/pipeline/enrichment.py:** Create `run_enrichment(adata, config, project_dir, progress_tracker=None) -> tuple[AnnData, dict]` (REQ-106):

1. Check for existing checkpoint.
2. Requires DE results in adata.uns["de_results"] (dict of {group_name: list[gene_dict]}). If not present, return (adata, {"stage": "enrichment", "status": "skipped", "reason": "No DE results found. Run differential expression first."}).
3. For each cell type in de_results:
   a. Extract significant upregulated genes: pvals_adj < 0.05 AND logfoldchanges > 0.5. Use the "names" field from each gene dict.
   b. If fewer than 5 significant genes, skip this cell type with status "too_few_genes".
   c. Run `gseapy.enrich(gene_list=sig_genes, gene_sets=config.enrichment_gene_sets, organism="human", outdir=None, cutoff=0.05)`.
   d. Store results as list of dicts from `enr.results.to_dict(orient="records")`.
   e. Wrap in try/except -- on failure, store {"error": str(e)} for that cell type.
4. Store enrichment results in adata.uns["enrichment_results"].
5. Save checkpoint.
6. Update progress.
7. Return (adata, enrichment_report) where report includes: stage="enrichment", gene_sets_used, n_cell_types_analyzed, n_cell_types_skipped, results_summary (per cell type: n_significant_terms or skip reason), timestamp.

Note: gseapy.enrich() may need network access to download gene set databases. Handle ConnectionError gracefully -- if gene set download fails, report the error per cell type without crashing the pipeline.

"""Disease-agnostic scRNA-seq analysis pipeline.

`run_pipeline(input_path, config, project_dir, task_manager=None, task_id=None) -> dict`:
1. Create StageProgressTracker(task_manager, task_id).
2. Check for resume: call `get_latest_checkpoint(project_dir)`. If a checkpoint exists, load it and skip all stages up to and including that checkpoint. Log which stages were skipped in the result.
3. Execute stages in order. Each stage follows the protocol: takes (adata, config, project_dir, progress_tracker) and returns (adata, report). Collect all reports in a results dict.

Stage sequence:
  a. **ingest**: `adata = ingest_data(input_path)`, update progress. Save checkpoint "ingest".
  b. **ambient_rna**: `adata, report = run_ambient_rna_removal(adata, config, project_dir, progress_tracker)`. If status is "skipped", log the warning in the audit context (store in results, don't call audit trail directly -- that's the caller's job).
  c. **qc**: `adata, report = run_qc(adata, config, project_dir, progress_tracker)`
  d. **processing**: `adata, report = run_processing(adata, config, project_dir, progress_tracker)`
  e. **annotation**: `adata, report = run_annotation(adata, config, project_dir, progress_tracker)`. After annotation, call `validate_annotations(adata)` from annotation.py and store discrepancies in results["annotation_validation"] (REQ-107).
  f. **de**: `adata, report = run_differential_expression(adata, config, project_dir, progress_tracker)`
  g. **enrichment**: `adata, report = run_enrichment(adata, config, project_dir, progress_tracker)`

4. Save final annotated adata to `{project_dir}/results/final.h5ad`.
5. Save combined results report to `{project_dir}/results/pipeline_report.json`.
6. Update progress to 1.0 (finalize).
7. Return results dict containing: all stage reports, resumed_from (stage name or None), final_h5ad_path, n_cells_final, n_genes_final, n_cell_types, pipeline_config (serialized), timestamp.

Resume logic detail: If `get_latest_checkpoint()` returns ("processing", path), then load that checkpoint adata, set resumed_from="processing", and start from the next stage (annotation). For stages before the checkpoint, store {"status": "skipped", "reason": "resumed_from_checkpoint"} in their report slots.

Export from __init__.py: run_pipeline, PipelineConfig, QCConfig, ProcessingConfig, TISSUE_DEFAULTS.
