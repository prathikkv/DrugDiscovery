**models.py** -- Define three Pydantic v2 BaseModel classes:

1. `SectionContent(BaseModel)`:
   - `title: str` -- section heading
   - `narrative: str = ""` -- AI-generated or template text
   - `data: dict = Field(default_factory=dict)` -- structured data for tables/charts
   - `charts: list[str] = Field(default_factory=list)` -- chart identifiers (keys into VisualizationBuilder output)

2. `DossierConfig(BaseModel)`:
   - `include_plotlyjs: bool | str = True` -- True for self-contained HTML, 'cdn' for smaller files. Default True per research pitfall #5 (pharma environments may be air-gapped).
   - `chart_width: int = 900`
   - `chart_height: int = 550`
   - `chart_scale: int = 2` -- retina/print quality
   - `brand_color: tuple[int, int, int] = (0, 113, 227)` -- #0071e3 design system color
   - `brand_name: str = "BioOrchestrator"` -- for headers/footers

3. `DossierData(BaseModel)`:
   - `gene_symbol: str`
   - `disease_context: str | None = None`
   - `generated_at: str = Field(default_factory=lambda: datetime.now(timezone.utc).isoformat())`
   - `scorecard: dict` -- from ScorecardResult.model_dump()
   - `comparative: dict | None = None` -- from ComparativeScorecard.model_dump() if multi-target
   - `evidence: dict` -- serialized AggregatedEvidence (results dict with source_name keys)
   - `reasoning: dict[str, dict] = Field(default_factory=dict)` -- mode_name -> ReasoningResult.model_dump()
   - `pipeline_report: dict | None = None` -- from pipeline_report.json if available
   - `gene_identifiers: dict | None = None` -- from GeneIdentifiers (canonical, ensembl, uniprot)
   - `sections: dict[str, SectionContent] = Field(default_factory=dict)` -- built by data_collector
   - `config: DossierConfig = Field(default_factory=DossierConfig)`

**data_collector.py** -- Create `collect_dossier_data()` function:

```python
def collect_dossier_data(
    scorecard_result: "ScorecardResult",
    evidence: "AggregatedEvidence",
    reasoning_results: dict[str, "ReasoningResult"] | None = None,
    pipeline_report: dict | None = None,
    comparative: "ComparativeScorecard | None" = None,
    config: DossierConfig | None = None,
) -> DossierData:
```

This function:
- Serializes ScorecardResult via `.model_dump()` into `scorecard` field
- Serializes AggregatedEvidence: build a dict with `gene` (GeneIdentifiers fields as dict), `disease_context`, and `results` (map source_name -> {confidence, data, error, is_fallback})
- Serializes each ReasoningResult (if provided) via `.model_dump()` keyed by mode name
- Copies pipeline_report dict as-is if provided
- Extracts gene_identifiers from evidence.gene into dict form
- Builds the `sections` dict with 7 SectionContent entries keyed as: "executive_summary", "target_overview", "evidence_dimensions", "ai_synthesis", "scorecard", "recommendations", "audit_trail"
- For each section, populate:
  - `title` with proper heading text
  - `narrative` with relevant data (e.g., executive_summary gets verdict rationale + synthesis summary; target_overview gets gene identifiers and protein function from UniProt evidence; evidence_dimensions gets dimension scores; ai_synthesis gets claims/summaries; scorecard gets composite breakdown; recommendations gets verdict-based next steps; audit_trail gets evidence hashes, model info, timestamps)
  - `data` with structured sub-data for tables (e.g., dimension score tables, sub-score breakdowns)
  - `charts` with chart identifier keys (e.g., executive_summary gets ["radar_single"])

Use lazy imports for upstream types (ScorecardResult, AggregatedEvidence, ReasoningResult) to avoid circular dependencies.

**visualizations.py** -- Create `VisualizationBuilder` class:

```python
class VisualizationBuilder:
    """Creates all Plotly figures for a dossier from DossierData."""

    def __init__(self, dossier_data: DossierData):
        self.data = dossier_data

    def build_all(self) -> dict[str, go.Figure]:
        """Build all available figures. Returns dict of chart_id -> Figure."""
```

The `build_all()` method should call individual builder methods and collect results into a dict. Each method returns a Figure or None (if data is missing). Chart IDs:

1. `radar_single` -- calls `build_single_radar()` from `src.scoring.comparative` using the scorecard data. Extract dimension scores and verdict from `self.data.scorecard`. Reconstruct enough data to call the existing function, OR re-implement a simple Scatterpolar inline if the upstream function requires the full Pydantic model (prefer calling existing code via lazy import).

2. `radar_comparative` -- calls `build_comparative_radar()` if `self.data.comparative` is not None.

3. `evidence_dimensions_bar` -- creates a horizontal bar chart (Plotly Bar) showing each dimension's score vs max_score. Use brand color for fill, light gray for unfilled. 7 bars for 7 dimensions. Add data_coverage annotation per dimension.

4. `score_breakdown` -- creates a stacked bar or grouped bar chart showing all sub-scores within each dimension. Use go.Bar with grouped barmode.

5. `umap_celltype` -- if pipeline_report has UMAP data, calls the existing `plot_umap()` from `bioorchestrator_real.utils.plotting` via lazy import. Pass coords dict and cell_type labels. Returns None if no pipeline data.

6. `expression_heatmap` -- if pipeline_report has expression matrix data, calls `plot_expression_heatmap()`. Returns None if missing.

7. `volcano` -- if pipeline_report has DE data, calls `plot_volcano()`. Returns None if missing.

For methods 5-7, use try/except with graceful return of None on ImportError or missing data. The dossier should work even without pipeline data (evidence-only assessment).

**chart_export.py** -- Create two utility functions:

```python
def export_chart_png(
    fig: go.Figure,
    output_path: Path,
    width: int = 900,
    height: int = 550,
    scale: int = 2,
) -> Path:
    """Export Plotly figure to PNG file. Returns output_path."""

def export_chart_svg(
    fig: go.Figure,
    output_path: Path,
    width: int = 900,
    height: int = 550,
) -> Path:
    """Export Plotly figure to SVG file. Returns output_path."""
```

Both functions call `fig.write_image()` with the appropriate format. PNG uses scale=2 for retina quality. SVG does not use scale parameter. Both ensure parent directory exists via `output_path.parent.mkdir(parents=True, exist_ok=True)`.

Add a `chart_to_png_bytes()` helper for the PDF renderer:
```python
def chart_to_png_bytes(
    fig: go.Figure,
    width: int = 900,
    height: int = 550,
    scale: int = 2,
) -> bytes:
    """Convert Plotly figure to PNG bytes (for in-memory PDF embedding)."""
    return fig.to_image(format="png", width=width, height=height, scale=scale)
```
