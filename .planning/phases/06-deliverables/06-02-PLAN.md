**html_renderer.py** -- Create `HTMLDossierRenderer` class:

```python
class HTMLDossierRenderer:
    """Renders DossierData as an interactive HTML file with embedded Plotly charts."""

    def __init__(self, config: DossierConfig | None = None):
        self.config = config or DossierConfig()
        self._env = Environment(
            loader=FileSystemLoader(Path(__file__).parent / "templates"),
            autoescape=select_autoescape(["html"]),
        )
        # Register custom filters
        self._env.filters["format_score"] = self._format_score
        self._env.filters["format_pct"] = self._format_pct
        self._env.filters["verdict_color"] = self._verdict_color

    def render(self, dossier_data: DossierData) -> str:
        """Render complete HTML dossier string."""

    def render_to_file(self, dossier_data: DossierData, output_path: Path) -> Path:
        """Render and write to file. Returns output_path."""
```

The `render()` method:
1. Creates a `VisualizationBuilder(dossier_data)` and calls `build_all()` to get all Plotly figures.
2. Converts each figure to an HTML div string using `fig.to_html(full_html=False, include_plotlyjs=...)`:
   - First chart: use `self.config.include_plotlyjs` (True or 'cdn')
   - Subsequent charts: `include_plotlyjs=False` to avoid loading plotly.js multiple times
   - Add `config={'displayModeBar': True, 'toImageButtonOptions': {'format': 'png', 'height': 600, 'width': 900, 'scale': 2}}` so users can download charts directly from HTML
3. Loads the `dossier.html` template and renders with context: `{"dossier": dossier_data, "charts": chart_divs_dict}`
4. Returns the rendered HTML string.

The `render_to_file()` method calls `render()` and writes to `output_path`, creating parent directories if needed. Returns the path.

Custom Jinja2 filters:
- `format_score`: `lambda v: f"{v:.1f}"` for numeric scores
- `format_pct`: `lambda v: f"{v*100:.0f}%"` for 0-1 fractions
- `verdict_color`: maps "GO" -> "#00b300", "CONDITIONAL" -> "#ff9900", "NO-GO" -> "#dd0000"
