1. `DossierPDF(FPDF)` -- fpdf2 subclass with branded headers and footers:

```python
class DossierPDF(FPDF):
    """Branded PDF with headers, footers, and page numbering."""

    def __init__(self, gene_symbol: str, disease_context: str = "", brand_color: tuple = (0, 113, 227)):
        super().__init__()
        self.gene_symbol = gene_symbol
        self.disease_context = disease_context
        self.brand_color = brand_color
        self.set_auto_page_break(auto=True, margin=25)

    def header(self):
        # Brand color line and title
        self.set_font("Helvetica", "B", 10)
        self.set_text_color(*self.brand_color)
        self.cell(0, 8, f"Target Assessment Dossier: {self.gene_symbol}", align="L")
        if self.disease_context:
            self.set_font("Helvetica", "", 8)
            self.set_text_color(128, 128, 128)
            self.cell(0, 8, f"  |  {self.disease_context}", align="R", new_x="LMARGIN")
        self.ln(4)
        self.set_draw_color(*self.brand_color)
        self.line(10, self.get_y(), 200, self.get_y())
        self.ln(8)

    def footer(self):
        self.set_y(-15)
        self.set_font("Helvetica", "I", 8)
        self.set_text_color(128, 128, 128)
        self.cell(0, 10, f"Page {self.page_no()}/{{nb}}", align="C")
```

2. `PDFDossierRenderer` -- main renderer class:

```python
class PDFDossierRenderer:
    """Renders DossierData as a professional PDF with static chart images."""

    def __init__(self, config: DossierConfig | None = None):
        self.config = config or DossierConfig()

    def render(self, dossier_data: DossierData) -> bytes:
        """Render complete PDF as bytes."""

    def render_to_file(self, dossier_data: DossierData, output_path: Path) -> Path:
        """Render and write to file. Returns output_path."""
```

The `render()` method:

1. Creates `DossierPDF(gene_symbol, disease_context, brand_color)` from config and dossier_data.
2. Calls `pdf.alias_nb_pages()` for total page count in footers.
3. Creates `VisualizationBuilder(dossier_data)` and calls `build_all()` to get figures.
4. Renders each section using dedicated private methods. Each section method:
   - Starts with `pdf.add_page()` (for major sections, or continues on current page for short ones)
   - Uses section data from `dossier_data.sections[section_key]`
   - Calls helper methods for common patterns (tables, charts, text blocks)

**Section rendering methods:**

`_render_executive_summary(pdf, dossier_data, figures)`:
- Section title "Executive Summary" in 16pt bold
- Verdict badge: colored rounded rectangle with GO/CONDITIONAL/NO-GO text
- Composite score displayed prominently
- Narrative text via multi_cell
- Strengths and risks as bullet points
- Radar chart embedded via `_embed_chart(pdf, figures.get("radar_single"))`

`_render_target_overview(pdf, dossier_data)`:
- Gene identifiers table (3 columns: Field, Value, Source)
- Protein function description via multi_cell
- Disease context

`_render_evidence_dimensions(pdf, dossier_data, figures)`:
- For each dimension in scorecard.composite.dimension_scores:
  - Dimension name as subsection heading
  - Score bar: colored rectangle proportional to score/max_score
  - Data coverage text
  - Sub-score table using `pdf.table()` with branded headings_style
  - Relevant narrative from evidence_dimensions section
- Evidence bar chart embedded

`_render_ai_synthesis(pdf, dossier_data)`:
- Full synthesis narrative
- Claims table: text, confidence, sources (using pdf.table with repeat_headings=1 for multi-page)
- Contradiction analysis if available
- Evidence gaps if available

`_render_scorecard(pdf, dossier_data, figures)`:
- Full scoring table with all dimensions and sub-scores
- Composite breakdown table (dimension, raw score, max, weight, weighted)
- Verdict rationale
- Score breakdown chart + comparative radar if available

`_render_recommendations(pdf, dossier_data)`:
- Verdict badge
- Key risks as numbered list
- Next steps as numbered list
- Data gaps as bullet list

`_render_audit_trail(pdf, dossier_data)`:
- Evidence source table (source, confidence, fetched_at, is_fallback)
- Evidence hash
- AI model and prompt version info
- Scoring formula version
- Generation timestamp

**Helper methods:**

`_embed_chart(pdf, fig, width=None)`:
- Uses `chart_to_png_bytes(fig, ...)` from chart_export module
- Wraps bytes in `io.BytesIO` (fresh instance per research pitfall #3)
- Calls `pdf.image(bytesio, w=width or pdf.epw)`

`_section_title(pdf, title)`:
- Sets font to Helvetica Bold 16pt
- Sets text color to brand color
- Writes title, adds 6pt spacing

`_subsection_title(pdf, title)`:
- Sets font to Helvetica Bold 12pt
- Writes title, adds 4pt spacing

`_body_text(pdf, text)`:
- Sets font to Helvetica 10pt
- Sets text color to dark (#1d1d1f)
- Uses multi_cell with line spacing

`_draw_verdict_badge(pdf, verdict_level)`:
- Colored rounded rectangle (green/orange/red)
- White bold text inside

`_draw_score_bar(pdf, score, max_score, width=100)`:
- Gray background bar, brand color fill proportional to score/max_score
- Score text overlay

All methods must handle missing data gracefully (check for None/empty before rendering). Use `pdf.table()` context manager with `headings_style=FontFace(emphasis="BOLD", color=(255,255,255), fill_color=brand_color)` for all tables. Use `cell_fill_color=245, cell_fill_mode="ROWS"` for alternating row colors.

The `render()` method calls `pdf.output()` which returns bytes. The `render_to_file()` method writes bytes to output_path.
