Create `src/evidence/cache.py` implementing `EvidenceCache` class:

Schema (CREATE TABLE IF NOT EXISTS evidence_cache):
- gene_symbol TEXT NOT NULL
- source_name TEXT NOT NULL
- fetched_at REAL NOT NULL (Unix timestamp)
- expires_at REAL NOT NULL (Unix timestamp)
- data_json TEXT NOT NULL
- PRIMARY KEY (gene_symbol, source_name)
- CREATE INDEX IF NOT EXISTS idx_cache_expires ON evidence_cache(expires_at)

Constructor: `__init__(self, db_path: Path = None, ttl_seconds: int = 86400)`. Default db_path from `config.EVIDENCE_CACHE_DB`. Initialize schema on first use via `get_connection()` from `src.db`.

Methods:
- `get(self, gene_symbol: str, source_name: str) -> EvidenceResult | None`: Query by PK. Return None if not found OR if `time.time() > expires_at`. Deserialize via `EvidenceResult.from_json()`.
- `get_stale(self, gene_symbol: str, source_name: str) -> EvidenceResult | None`: Query by PK ignoring expiry. Returns the cached entry even if expired (stale). Returns None only if no entry exists at all. Used as fallback when live fetch fails after retry exhaustion (REQ-210 step 2). Deserialize via `EvidenceResult.from_json()` and set `is_fallback=True` on the returned result.
- `put(self, gene_symbol: str, source_name: str, result: EvidenceResult) -> None`: INSERT OR REPLACE with current time and ttl. Only cache results with `confidence > 0.0` (do NOT cache error results per research anti-pattern guidance). Serialize via `result.to_json()`.
- `invalidate(self, gene_symbol: str = None, source_name: str = None) -> int`: Delete entries matching filters. If both None, delete all. Return rowcount. Use BEGIN IMMEDIATE for write atomicity (per project pattern from 01-01).
- `cleanup_expired(self) -> int`: Delete all rows where `expires_at < time.time()`. Return count deleted.

Follow existing patterns: per-operation connections (get fresh connection, try/finally close), BEGIN IMMEDIATE for writes.
  </action>

  <action>
Create `src/evidence/gene_resolver.py` implementing `GeneResolver` class:

Module-level constant `LOCAL_ALIASES` dict mapping common aliases (case-normalized to uppercase keys):
- PD-L1 -> CD274, PD-1 -> PDCD1, HER2 -> ERBB2, HER3 -> ERBB3
- P53 -> TP53, VEGF -> VEGFA, TNF-ALPHA -> TNF, IL-6R -> IL6R
- CTLA-4 -> CTLA4, GIPR -> GIPR, GLP1R -> GLP1R

Constructor: `__init__(self)`. Initialize `mygene.MyGeneInfo()` instance and an in-memory cache dict `_cache: dict[str, GeneIdentifiers]`.

Method `resolve(self, symbol: str) -> GeneIdentifiers`:
1. Normalize input: `symbol.strip()` (preserve case for MyGene.info queries but use upper for alias lookup)
2. Check `_cache` by uppercase key -- return immediately if found
3. Check `LOCAL_ALIASES` by uppercase key -- if found, use canonical as the query symbol for MyGene.info
4. Query MyGene.info: `self.mg.query(query_symbol, scopes="symbol,alias", species="human", fields="symbol,ensembl.gene,uniprot.Swiss-Prot", size=1)`
5. Parse response: extract `symbol` (canonical), `ensembl.gene` (Ensembl ID -- may be string or list, take first), `uniprot.Swiss-Prot` (UniProt accession -- may be string or list, take first)
6. If MyGene.info fails (network error, empty results): construct GeneIdentifiers with canonical_symbol=query_symbol, ensembl_id=None, uniprot_accession=None
7. Store in `_cache` and return

Wrap MyGene.info call in try/except (catch all exceptions) -- gene resolution must never crash the pipeline. Log warnings using Python `logging` module.

Important: The `fetch()` signature in interface.py takes `GeneIdentifiers` (not raw symbol), so the resolver runs BEFORE sources are called. This means OpenTargets gets its Ensembl ID and ChEMBL gets its UniProt accession directly from the resolver output.
