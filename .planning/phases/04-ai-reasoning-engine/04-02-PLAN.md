**src/reasoning/fallback.py** -- LLM provider fallback chain (REQ-306):

Create `FallbackChain` class that wraps the existing providers from `bioorchestrator_real/utils/llm_provider.py`:

1. `__init__(self, audit_trail: AuditTrail | None = None, ollama_model: str = "qwen3:8b", groq_model: str = "llama-3.3-70b-versatile", anthropic_model: str = "claude-sonnet-4-20250514")`:
   - Build provider chain by checking availability in order:
     a. Ollama: check `OllamaProvider.is_available()` -- if yes, add `OllamaProvider(model=ollama_model)`
     b. Groq: check `os.environ.get("GROQ_API_KEY")` -- if key exists, add `GroqProvider(api_key=key, model=groq_model)`
     c. Anthropic: check `os.environ.get("ANTHROPIC_API_KEY")` -- if key exists, add `AnthropicProvider(api_key=key, model=anthropic_model)`
   - Store `self.providers: list`, `self.audit: AuditTrail | None`
   - If no providers available, log a warning but do NOT raise (allow construction -- will fail at call time)

2. `get_provider(self) -> tuple[object, str]`: Return `(provider, provider.name)` for the first available provider. Raise `RuntimeError("No LLM provider available")` if list is empty.

3. `execute_with_fallback(self, fn: Callable, *args, **kwargs) -> Any`:
   - Iterate through providers. For each, call `fn(provider, *args, **kwargs)`.
   - On success, return result.
   - On exception: log the error, record a fallback event in audit trail using `self.audit.append_record(user_id="system", action="llm_fallback", resource_type="ai_reasoning", resource_id=provider.name, details={"error": str(e), "from_provider": provider.name, "to_provider": next_provider_name_or_"none"})`.
   - Collect `(provider_name, error_str)` tuples in a `fallback_events` list. Store on `self.last_fallback_events` for the provenance tracker to read.
   - If all providers fail, raise `RuntimeError(f"All LLM providers failed: {errors}")`.

4. Property `available_providers -> list[str]`: Return list of provider names.

IMPORTANT per research Pitfall 4: Do NOT switch providers mid-loop. If a provider fails during a tool-calling round, restart the ENTIRE reasoning on the next provider.

**src/reasoning/tool_loop.py** -- Agentic tool-calling loop with 10-round limit (REQ-302):

Create `run_tool_loop` function following the pattern from `bioorchestrator_real/utils/ai_copilot.py` but adapted for the reasoning engine:

```python
def run_tool_loop(
    provider,
    system: str,
    initial_message: str,
    tools: list[dict],
    tool_executor: ToolExecutor,
    token_manager: TokenManager | None = None,
    max_rounds: int = 10,
) -> ToolTrace:
```

Implementation:
1. Build initial messages list: `[{"role": "user", "content": initial_message}]`
2. Call `provider.chat_with_tools(system, messages, tools)` to get the first LLM response
3. Create a `ToolTrace()` and add the first response to `trace.rounds`
4. Enter a loop for up to `max_rounds` iterations:
   a. Check `result["stop_reason"]` -- if not `"tool_use"` or no `result["tool_calls"]`, break (LLM is done calling tools)
   b. For each tool call in `result["tool_calls"]`:
      - Record start time
      - Call `tool_executor.execute(tc["name"], tc["arguments"])`
      - Record end time, compute duration_ms
      - Create `ToolCallRecord(name=tc["name"], arguments=tc["arguments"], result=output, round_number=current_round, duration_ms=duration_ms)`
      - Add to `trace.tool_calls`
   c. Build tool result messages for the provider. Use the Anthropic-style format that `llm_provider.py` expects for cross-provider compatibility:
      - Append `{"role": "assistant", "content": result["raw_content"]}` to messages
      - Append `{"role": "user", "content": [{"type": "tool_result", "tool_use_id": tc["id"], "content": json.dumps(output, default=str)} for tc in result["tool_calls"]]}` to messages
   d. If `token_manager` is provided, check `token_manager.fits_context(system, messages)`. If it doesn't fit, log a warning and break the loop early (context exhausted).
   e. Call `provider.chat_with_tools(system, messages, tools)` for the next round
   f. Add response to `trace.rounds`, increment round counter
5. After the loop, set `trace.final_text = result["text"]` and `trace.total_rounds = round_counter`
6. Return the `ToolTrace`

CRITICAL: Use `json.dumps(output, default=str)` when serializing tool results to handle datetime objects, Path objects, etc.

**src/reasoning/provenance.py** -- Provenance tracking and evidence hashing (REQ-305):

Create `ProvenanceTracker` class:

1. `__init__(self, audit_trail: AuditTrail | None = None)`:
   - Store audit_trail reference
   - Import hashlib for evidence hashing

2. `hash_evidence(self, evidence_data: dict | str) -> str`:
   - If dict, serialize with `json.dumps(data, sort_keys=True, separators=(",", ":"), default=str)`
   - Compute SHA256 hex digest
   - Return the hash string

3. `hash_aggregated_evidence(self, aggregated: AggregatedEvidence) -> dict[str, str]`:
   - For each source_name -> EvidenceResult in `aggregated.results`:
     - Hash the result's data dict via `hash_evidence(result.to_json())`
   - Return `{source_name: sha256_hash}` dict

4. `build_provenance(self, model_name: str, provider_name: str, prompt_version: str, input_evidence_hashes: dict[str, str], tool_trace: ToolTrace, fallback_events: list[dict] | None = None) -> ProvenanceRecord`:
   - Create a `ProvenanceRecord` with all fields populated:
     - `tools_used`: from `tool_trace.tools_used()`
     - `tool_rounds`: from `tool_trace.total_rounds`
     - `trace_id`: generate via `str(uuid.uuid4())`
     - `timestamp`: `datetime.now(timezone.utc).isoformat()`
   - Return the record

5. `record_to_audit(self, provenance: ProvenanceRecord, gene_symbol: str, mode: str, user_id: str = "system") -> str | None`:
   - If `self.audit_trail` is None, return None
   - Call `self.audit_trail.append_record(user_id=user_id, action="ai_reasoning", resource_type="ai_reasoning", resource_id=f"{gene_symbol}:{mode}", details=provenance.to_audit_details())`
   - Return the audit record hash

6. `save_trace(self, trace_id: str, tool_trace: ToolTrace, output_dir: Path | None = None) -> Path | None`:
   - If output_dir is None, use `Path("data/reasoning_traces")`
   - Create directory if it doesn't exist
   - Save `tool_trace.model_dump()` as JSON to `output_dir / f"{trace_id}.json"` using `json.dumps(..., default=str, indent=2)`
   - Return the file path
   - Per research recommendation: store full traces in files, not in the audit trail details_json (which should stay lean)
