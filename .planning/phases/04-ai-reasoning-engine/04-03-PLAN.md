**src/reasoning/hallucination.py** -- Citation validation and source counting (REQ-307):

1. `check_citations(text: str, tool_trace: ToolTrace) -> list[dict]`:
   - Extract all `[Source: X]` citations from text using `re.findall(r'\[Source:\s*(\w+)\]', text)`
   - Get set of tools actually called from `tool_trace.tools_used()`
   - Import `CITATION_TO_TOOL` from `src.reasoning.tools`
   - For each citation, check if the mapped tool was actually called. If not, append `{"type": "phantom_citation", "citation": citation, "detail": f"Cites {citation} but tool {tool_name} was never called"}`
   - Check for uncited substantive paragraphs: split text by `\n\n`, for each paragraph >100 chars without any `[Source:` reference, append `{"type": "uncited_claim", "detail": f"Paragraph without citation: {para[:80]}..."}`
   - Return the list of issues

2. `check_confidence_sources(claims: list[Claim]) -> list[dict]`:
   - For each claim with `confidence > 0.8`:
     - If `len(claim.sources) < 3`, append `{"type": "insufficient_sources", "claim": claim.text[:80], "confidence": claim.confidence, "sources_found": len(claim.sources), "sources_required": 3}`
   - Return the list of issues

3. `validate_output(text: str, claims: list[Claim], tool_trace: ToolTrace) -> list[dict]`:
   - Convenience function that calls both `check_citations` and `check_confidence_sources`
   - Returns combined list of all issues

**src/reasoning/engine.py** -- Multi-mode reasoning orchestrator (REQ-301):

Create `ReasoningEngine` class:

1. `__init__(self, fallback_chain: FallbackChain | None = None, tool_executor: ToolExecutor | None = None, prompt_registry: PromptRegistry | None = None, provenance_tracker: ProvenanceTracker | None = None, token_manager: TokenManager | None = None, audit_trail: AuditTrail | None = None)`:
   - Create defaults for any None parameters: `FallbackChain(audit_trail=audit_trail)`, `ToolExecutor()`, `PromptRegistry()`, `ProvenanceTracker(audit_trail=audit_trail)`, `TokenManager()`
   - Store all as instance attributes

2. `reason(self, gene_symbol: str, mode: ReasoningMode, disease_context: str | None = None, evidence: AggregatedEvidence | None = None, user_id: str = "system") -> ReasoningResult`:

   This is the main entry point. Steps:

   a. **Get versioned prompt**: `system_prompt, prompt_hash = self.prompt_registry.get(mode)`

   b. **Build user message**: Construct a message string like:
      ```
      Analyze the gene {gene_symbol}{' in the context of ' + disease_context if disease_context else ''}.
      Use the available tools to query data sources and {mode-specific instruction}.
      Structure your response with clear claims, each citing its source using [Source: SourceName] format.
      ```
      The mode-specific instruction varies:
      - HYPOTHESIS: "generate testable hypotheses"
      - SYNTHESIS: "synthesize findings across all sources"
      - CONTRADICTION: "identify contradictions between sources"
      - GAP: "identify gaps in the available evidence"
      - CONFIDENCE: "assess confidence in the therapeutic potential"

   c. **Hash evidence if provided**: If `evidence` is not None, call `self.provenance_tracker.hash_aggregated_evidence(evidence)` to get input_evidence_hashes. Also include a brief evidence summary in the user message if evidence data is available (append key findings from each source that had confidence > 0).

   d. **Check token budget**: If evidence summary + system prompt exceed the token budget, use `self.token_manager` to check and potentially flag for summarization. If `token_manager.should_summarize(evidence_summary_text)`, truncate the evidence summary to fit.

   e. **Run tool loop with fallback**: Use `self.fallback_chain.execute_with_fallback()`:
      - The function to execute takes a provider and runs `run_tool_loop(provider, system_prompt, user_message, TOOL_DEFINITIONS, self.tool_executor, self.token_manager, max_rounds=10)`
      - Returns the ToolTrace
      - If all providers fail, create a ReasoningResult with empty claims and the error in summary

   f. **Parse output**: Extract the final text from the ToolTrace. Attempt to parse structured claims from the LLM output:
      - Use a simple heuristic parser: look for numbered items (e.g., "1.", "2.") or bullet points, extract text and `[Source: X]` citations per item
      - Create `Claim` objects with `text`, `sources` (from citations), and `confidence` (default 0.5 unless the LLM explicitly states a confidence value)
      - This parser does NOT need to be perfect -- it extracts what it can. The raw output is always preserved.

   g. **Hallucination check**: Call `validate_output(trace.final_text, claims, trace)` to get issues list

   h. **Build provenance**: Call `self.provenance_tracker.build_provenance(model_name, provider_name, prompt_hash, input_evidence_hashes, trace, fallback_events)`
      - Get model_name and provider_name from `self.fallback_chain.get_provider()` (or from the last successful provider tracked during fallback)
      - Record to audit trail: `self.provenance_tracker.record_to_audit(provenance, gene_symbol, mode.value, user_id)`
      - Save trace: `self.provenance_tracker.save_trace(provenance.trace_id, trace)`

   i. **Return ReasoningResult**: Build and return with all fields populated:
      - mode, gene_symbol, disease_context, claims, summary=trace.final_text, raw_output=trace.final_text, tool_trace=trace, hallucination_issues=issues

3. `reason_all_modes(self, gene_symbol: str, disease_context: str | None = None, evidence: AggregatedEvidence | None = None, user_id: str = "system") -> dict[ReasoningMode, ReasoningResult]`:
   - Convenience method that runs `reason()` for each of the 5 modes
   - Returns a dict mapping mode -> result
   - Catches exceptions per mode (one mode failing shouldn't prevent others)

Update `src/reasoning/__init__.py` to export `ReasoningEngine`, `check_citations`, `check_confidence_sources`, `validate_output`.
