**src/reasoning/models.py** -- Pydantic models and enums for the reasoning subsystem:

1. `ReasoningMode` enum (str, Enum) with 5 values:
   - `HYPOTHESIS = "hypothesis"` -- Generate testable hypotheses from evidence
   - `SYNTHESIS = "synthesis"` -- Synthesize findings across sources
   - `CONTRADICTION = "contradiction"` -- Identify conflicting evidence
   - `GAP = "gap"` -- Flag missing evidence and data gaps
   - `CONFIDENCE = "confidence"` -- Assess confidence levels with justification

2. `ToolCallRecord` (Pydantic BaseModel):
   - `name: str` -- Tool function name (e.g., "query_opentargets")
   - `arguments: dict` -- Arguments passed to the tool
   - `result: dict | str | None` -- Tool execution result
   - `round_number: int` -- Which round of the tool loop (1-10)
   - `duration_ms: float = 0.0` -- Execution time in milliseconds

3. `ToolTrace` (Pydantic BaseModel):
   - `rounds: list[dict]` -- Raw LLM response per round (role/content/tool_calls)
   - `tool_calls: list[ToolCallRecord]` -- All tool calls executed across all rounds
   - `total_rounds: int = 0` -- Total rounds completed
   - `final_text: str = ""` -- Final LLM text output after all rounds
   - Method: `tools_used() -> list[str]` returning unique tool names from tool_calls

4. `Claim` (Pydantic BaseModel):
   - `text: str` -- The claim text
   - `confidence: float` -- Confidence score (0.0-1.0)
   - `sources: list[str]` -- Source citations (e.g., ["OpenTargets", "PubMed"])
   - `evidence_snippets: list[str] = []` -- Supporting evidence text fragments

5. `ReasoningResult` (Pydantic BaseModel):
   - `mode: ReasoningMode` -- Which reasoning mode was used
   - `gene_symbol: str` -- Target gene analyzed
   - `disease_context: str | None = None` -- Optional disease context
   - `claims: list[Claim] = []` -- Structured claims with citations
   - `summary: str = ""` -- Overall reasoning summary text
   - `raw_output: str = ""` -- Raw LLM output text
   - `tool_trace: ToolTrace | None = None` -- Full tool-calling trace
   - `hallucination_issues: list[dict] = []` -- Issues found by hallucination checker
   - `created_at: str` -- ISO 8601 timestamp (use default_factory with datetime.now(timezone.utc).isoformat())

6. `ProvenanceRecord` (Pydantic BaseModel):
   - `model_name: str` -- e.g., "qwen3:8b"
   - `provider_name: str` -- e.g., "Ollama", "Groq", "Claude"
   - `prompt_version: str` -- SHA256 hash of the system prompt used
   - `input_evidence_hashes: dict[str, str] = {}` -- {source_name: sha256_of_evidence_data}
   - `tools_used: list[str] = []` -- Tool names invoked during reasoning
   - `tool_rounds: int = 0` -- Number of tool-calling rounds completed
   - `fallback_events: list[dict] = []` -- [{provider, error, timestamp}]
   - `trace_id: str` -- UUID for referencing full trace in storage
   - `timestamp: str` -- ISO 8601 timestamp
   - Method: `to_audit_details() -> dict` that returns a dict suitable for `AuditTrail.append_record(details=...)`, including model, provider, prompt_version, input_evidence_hashes, tools_used, tool_rounds, fallback_events, and trace_id (but NOT the full reasoning chain -- that is stored separately per research recommendation)

**src/reasoning/prompts.py** -- Versioned system prompt registry with SHA256 hashing (REQ-304):

1. Define `PROMPT_TEMPLATES: dict[ReasoningMode, str]` with 5 mode-specific system prompts. Each prompt must:
   - Clearly state the mode's analytical purpose
   - Instruct the LLM to use tools to gather data before making claims
   - Require `[Source: ToolName]` citation format for every claim
   - Specify the expected output structure (claims with confidence scores and citations)
   - Be disease-agnostic (no hardcoded biology)

   Prompt content guidance by mode:
   - **HYPOTHESIS**: "You are a scientific hypothesis generator. Query available data sources to generate testable hypotheses about the target gene. Each hypothesis must be grounded in tool results. Format: numbered hypotheses with evidence citations."
   - **SYNTHESIS**: "You are a scientific evidence synthesizer. Query all relevant data sources and synthesize findings into a coherent narrative. Identify patterns across sources. Every statement must cite its source."
   - **CONTRADICTION**: "You are a scientific evidence auditor focused on identifying contradictions. Query multiple sources and flag where evidence conflicts. For each contradiction, cite the conflicting sources."
   - **GAP**: "You are a scientific evidence gap analyst. Query available sources and identify what evidence is missing, incomplete, or insufficient. Flag areas where more data is needed."
   - **CONFIDENCE**: "You are a scientific confidence assessor. Query all sources and rate the overall confidence in the target gene's therapeutic potential. Claims with confidence >0.8 must cite 3+ independent sources."

   Each prompt should be 200-400 words, covering: role, available tools, citation requirements, output format.

2. `PromptRegistry` class:
   - `__init__()`: Builds `_prompts: dict[ReasoningMode, tuple[str, str]]` mapping mode -> (text, sha256_hash). Compute hash via `hashlib.sha256(text.encode("utf-8")).hexdigest()`.
   - `get(mode: ReasoningMode) -> tuple[str, str]`: Returns (prompt_text, sha256_hash).
   - `register(mode: ReasoningMode, text: str) -> str`: Register a custom prompt, returns its SHA256 hash.
   - `get_all_versions() -> dict[str, str]`: Returns {mode_value: sha256_hash} for all modes.

**src/reasoning/token_manager.py** -- Token counting and context window management (REQ-308):

1. `TokenManager` class:
   - `__init__(max_context: int = 8192, output_reserve: int = 2048)`:
     - Store `max_context`, `output_reserve`, compute `available = max_context - output_reserve`
     - Lazy-load tiktoken: try `import tiktoken; self._enc = tiktoken.get_encoding("cl100k_base")` in `__init__`. If tiktoken is not installed, set `self._enc = None` and fall back to `len(text) // 4` approximation (with a logged warning on first use).
   - `count_tokens(text: str) -> int`: Use tiktoken if available, else `len(text) // 4`.
   - `should_summarize(evidence_text: str, threshold: int = 8000) -> bool`: Returns True if token count exceeds threshold.
   - `fits_context(system: str, messages: list[dict], buffer: int = 500) -> bool`: Estimate whether the current conversation fits in context by counting tokens across system + all message content, comparing to `self.available - buffer`.
   - `truncate_tool_result(result_text: str, max_tokens: int = 500) -> str`: If result exceeds max_tokens, truncate to fit by taking the first N characters (estimated via token count) and appending "... [truncated]".
